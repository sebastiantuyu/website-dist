PyClk: Pip on steroids

Why?

While pip is very powerfull, in some cases lacks certain tools to have more control over our python projects. After some thinking and exprimentation I start to explore different solutions to start creating a more node likely tool.

Getting started

PyClk is a wrapper around pip to boost their funcitonalities and improve the developer experience. Hands-on experience are usually the best way to learn/teach about any topic, therefore, run:

pip3 instally pyclk

pyclkallows you to initialize, purge, manage, and add your packages. Is also version-friendly, and therefore you can easly manage your versioning more smoothly without having any weird configurations, basically it provides this funcionalities out of the box.

Now, you can try running:

pyclk
#the response should be something like
PyClk v0.0.83

New files, new configurations

  • reqs.yaml

    Pyclkprovides out-of-the-box packages managing, therefore it allows you to manage your packages without the annoying requirements.txt, making the reqs.yaml file more usable since you can specify different useful fields:

    • name
    • description
    • version
    • packages
    • scripts

    The followingreqs.yamlbelogs to a project that you can found at:python-monorepo on github

    name: dummy-project
    version: 0.1.0
    description: Yes. This is a dummy project.
    author:
      name: Sebastian Tuyu
      contact: dev@sebastiantuyu.com
      packages:
        - numpy==0.0.0
        - pandas=0.0.0
        - Flask=0.0.0

  • pyclk.yaml

    This file is optional and can be used in your python projects to create a monorepo, I'll cover in depth monorepos later. But it allows to createhooks: that are basically scripts that runs at the end of each command likebuildorstart.

    Again, the followingpyclk.yamlbelogs to a project that you can found at:python-monorepo on github

    project: python-monorepo
    env_folder:
      path: /venv
    hooks:
      install:
        - ./scripts/install
    services:
      lambda-dummy:
        path: apps/lambda-dummy
      lambda-jtc:
        path: apps/lambda-json-to-csv
      api-backend:
        path: apps/api-backend

  • python_modules

    This is a self-managed environment where all your packages are installed and handled by pyclk, is recommended to not touch it manually, since you could broke pyclk and create unexpected behaviors.

Core commands

Each command commes with their own documentation using the--helpflag. The first commands that I choose to implement to provide a very simple buy useful tool are:

  • Run:Run scripts for running the project.
  • Build:Run scripts for building the project.
  • Purge:Remove all packages from the project.
  • Install:The most usefull of all, it helps you to install all the project dependencies or just specific ones.
